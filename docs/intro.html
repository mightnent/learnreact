Intro

Component

create a folder inside src and call it components. Basically where all our reusable code lives

rmb to export. If export default, then naming doesn't matter in import

You can also use a class based component, however, that is slight more complicated. So functional ones are first choice

//component
import React from 'react';

function Header() {
    return <h1>This is Header</h1>
}

export {Header};
//app.js
import './App.css';
import { Header } from './component/header';

function App() {
  return (
    <div className="App">
      <Header/>
    </div>
  );
}

export default App;

  );

export default App;
//footer.js
import React, {Component} from 'react';

class Footer extends Component{
    render(){
        return(
            <React.Fragment>
								{/* myAlert comes from the app.js attribute */}
                <h2 onClick={this.props.myAlert}>
                    This is a footer
                </h2>
            </React.Fragment>
        )
    }
}

export {Footer};

State

only class based component can have state

Rationale: props are immutable, so you want to have state in each component

import React, {Component} from 'react';

class Footer extends Component{

    state={
        name:"Mike"
    }

    //arrow function : avoid "this" binding
    changed = event =>{
        this.setState({name:event.target.value});
        console.log(this.state);
    }

    render(){
        return(
            <React.Fragment>
 
                <h2 onClick={this.props.myAlert}>
                    This is a footer
                </h2>
                <input 
                    value={this.state.name}
                    onChange={this.changed}
                    type="text"
                />
          
            </React.Fragment>
        )
    }
}

export {Footer};

Context

The prop system cannot pass attributes to many different components efficiently. Hence, you can use context to sort of make some attribute value "global"

//snippet of index.js
import {Footer} from './component/footer';
import reportWebVitals from './reportWebVitals';
import {Route,BrowserRouter} from 'react-router-dom';

const context = React.createContext();
//note: the naming must start with caps
export const CtxConsumer = context.Consumer;

const animals_arr = ['bird','bat','cat'];

const routing = (
  <BrowserRouter>
    <context.Provider value={{animals:animals_arr}}>
      <div>
        <Route exact path="/" component={App}></Route>
        <Route path="/footer" component={Footer}></Route>
      </div>
    </context.Provider>
  </BrowserRouter>
)

When you pass the animals_arr into the context, it will be made available to everyone wrapped in the context.Provider tag. Note: You need to export the context consumer so the other components can use it.

so for instance, in the footer component

Hooks

useState

import React, {useState} from 'react';

const Numbers = () =>{
		//this is by convention, you have a getter and setter
    const [numbers,setNumbers] = useState(['1','2','3']);

    const addNum = () =>{
				//...numbers means the entire numbers array retrieved by getter
        setNumbers([...numbers,'4']);
    }

    return(
        <div>
            <h1>Numbers</h1>
						{/* the {} just means injecting js into html */}
            {numbers.map(num=>{
                return <h2>{num}</h2>
            })}
            <button onClick={addNum}>Add Number to list</button>
        </div>
    )
}

export default Numbers;

useEffect

import React, {useState,useEffect} from 'react';

const Numbers = () =>{

    const [numbers,setNumbers] = useState(['1','2','3']);

    const addNum = () =>{
        setNumbers([...numbers,'4']);
    }

    useEffect(() => {
        console.log("caught");
    }, [addNum])

    return(
        <div>
            <h1>Numbers</h1>
            {/* the {} just means injecting js into html */}
            {numbers.map(num=>{
                return <h2>{num}</h2>
            })}
            <button onClick={addNum}>Add Number to list</button>
        </div>
    )
}

export default Numbers;

It is used to catch events. It is powerful because it can catch any desired changes and trigger functions to respond to them
